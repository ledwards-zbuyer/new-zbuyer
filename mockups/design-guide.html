Seller Experience — Design Guide

The visual system behind the seller homepage, DR lander, lead funnel and Cash Value Report. Everything on this page is rendered with the exact tokens and CSS the live pages use — if it looks right here, it's right. Companion doc for form/funnel behavior: FORM_DESIGNER.md in the repo.

Typography

One typeface: Inter (Google Fonts, weights 400–900). Hierarchy comes from weight and tracking, not from more fonts. Big text tightens (negative letter-spacing), tiny labels widen and uppercase.

Shadows are soft, long-throw, and always negative-spread — depth without gray halos. Vertical rhythm: report page stacks cards at 16px gap; card padding 22px (26–30 for modals); control rows gap 10px; mobile page gutters 16–20px.

Components

Buttons

Primary (orange): weight 800, 12px radius, arrow suffix on funnel-advancing actions. One per view — every equal-weight action beside it is also orange only when the actions are true peers (Call / Text on the report). Outline (white/blue): secondary choices. Quiet link: tertiary, underlined blue. Escape hatches ("Do not contact me", "No thanks") are quiet links, never gray buttons — visible, not promoted.

Components

Chips & choices

Chips are the default answer control: one-word, self-ordering options in a single row (equal flex:1 widths; a deliberately narrower + lighter "No" sits last). Selected = --tint fill + --c1 border/text. Suggestion chips (SomethingSpecial) pack left-aligned in a horizontally scrolling carousel — right-edge fade hints at more, touch scrolls natively, mouse drag-scrolls on desktop. Validation: the chip row shakes vertically and an error line pops — color alone never carries the error.

Components

Inputs & the address box

Please enter your phone number.

Text inputs: 16px font (prevents iOS zoom), 1.5px --line border, 12px radius, focus = border flips to --c1 (no glow). Desktop hero address input is borderless inside its white card; mobile styles it as a --soft filled box. A verified address collapses to a two-line chip (street bold / "City, ST zip" muted small) and the card shrinks to fit; tapping the chip re-opens editing, where a circled-X clears the whole line in one tap. Street View strip (when enabled) sits above, full card width, fixed height — it never widens the card.

Components

Sliders

Two flavors, one language: tall pill handles with a 3px white border and a real shadow — unmistakably grabbable. Funnel repairs dial: 16×44, blue. Report value slider: 26×70, orange, grip ridges. Both slide freely and snap to stops on release. iOS gotcha: the visible handle is a custom element positioned by JS (iOS ignores native thumb styling); pointer events + touch-action:none make press-drag work on the whole track.

Alert notices: #FFF4F0 fill, 2px --cta border, icon coin, dismissible × (dismissal persists for the session). Trust stats: number in --c1 800, label muted. Eyebrows introduce sections everywhere: caps, tracked out, dotted.

Behavior

Motion & voice

Motion: entrances 200–300ms ease with slight rise; hero "cash value" shimmer loops gently (4s); errors shake ~300ms; the Z-beat interstitial (two triangles walking in, spinning as a rigid pair into the Z) covers API round-trips between funnel steps. Every animation has a prefers-reduced-motion fallback — no exceptions.

Voice: plain, confident, second person. Benefit first ("Get my cash value report"), never mechanics first ("Submit"). Questions are one line; answers are one word where possible. Never put "I'm selling" in the user's mouth — frame around the report they're already getting. Optional means optional: say "totally optional" and mean it (no validation). Settled facts read as settled ("A local expert will be in touch"), not as questions.

Don'tBlue CTA buttons; two orange CTAs doing different jobs; gray disabled-looking escape links; ALL-CAPS body copy; pure #000 text; sharp corners; error color as decoration.
DoOne orange action per view; chips over dropdowns; weight over size for emphasis; soft long shadows; 16px+ input text; quiet-link escape hatches; reduced-motion fallbacks.
